# 소개글 작성용 자료: "과제 채점 도우미"

> 이 파일은 앱을 소개하는 글(동료 교사 안내문, 블로그, 발표자료 등)을 쓸 때 AI에게 맥락으로 주기 위한
> 사실 자료입니다. claude.ai 등에 이 파일을 첨부하고 "이 자료로 ○○용 소개글을 써줘"라고 요청하세요.
> API 키 등 비밀정보는 들어 있지 않습니다.

## 무엇인가

한국 중·고등학교 교사를 위한 웹앱. 학생이 수행평가 과제를 제출하면 교사가 등록한 채점 기준에 따라
AI가 채점·피드백·"과목별 세부능력 및 특기사항(세특)" 초안을 만들어 주고, 교사가 검토·수정해 학생에게
공개한다. 만든 사람은 고등학교 물리 교사.

## 해결하려는 문제

수행평가 채점, 학생별 피드백 작성, 세특 작성은 교사 업무 중 부담이 큰 편이다. 특히 세특은 학생 수만큼
써야 하고 NEIS 입력 규정도 지켜야 한다.

## 주요 기능

1. **과제 제출 + AI 채점**
   - 학생이 pdf/docx/hwpx/txt 제출 → 교사가 채점 기준 등록 → AI가 점수, 학생용 피드백, 세특 초안,
     "채점 근거"(교사 전용)를 생성 → 교사가 검토·수정 후 공개
   - PDF는 텍스트만 뽑지 않고 원본을 통째로 AI에 보내 그림·표·그래프·손글씨까지 채점에 반영
   - 세특은 NEIS 규정을 반영해 수식·영어를 쓰지 않고, 3인칭 관찰형("~함")으로 작성. 역량어를 결론처럼
     붙이지 않고 수업 장면 중심으로 서술

2. **평가계획 가져오기**
   - 학기 평가계획 PDF를 올리면 AI가 수행평가 항목별로 분리하고 성취기준·평가요소·배점·채점기준·
     유의점을 구조화해 추출 → 교사가 검토·수정·병합 → 여러 클래스에 과제 일괄 생성
   - 채점 기준은 영역별·배점 단계별로 줄바꿈 정리되어 읽기 쉽게 들어감

3. **적응형 문제풀이(CAT)**
   - 클래스별 문제은행. 문항은 직접 입력하거나 문제지 PDF에서 AI가 일괄 추출(수식은 LaTeX로,
     그림은 PDF에서 드래그해 잘라 붙임)
   - 학생이 틀리면 같은 개념 문항이 우선 출제되고 난이도가 조정됨
   - 한 회차에 푼 문항은 다시 안 나오고, 복습 후 "다시 도전"에서 재도전
   - 피드백은 정답을 알려주지 않고 개념 설명과 힌트만 준다

4. **학생 대시보드** — 공지, 복습이 필요한 개념(개념별 AI 설명), 선생님 피드백

5. **교사 대시보드** — 검토 대기 제출물, 마감 임박 과제(제출 현황), 클래스별 정답률,
   학습 지원이 필요한 학생

6. **클래스 학습 분석** — 개념·유형·난이도별 통과율 표 + AI가 학급의 약점과 지도 방향 분석

7. **관리자** — 교사 계정 발급(임시 비밀번호 1회 표시). 교사 자율가입은 막혀 있음

## 기술

- Next.js(App Router) + Supabase(Postgres/Auth/Storage) + Google Gemini API, Vercel 배포
- 학교마다 각자 배포해서 쓰는 구조 → 학생 데이터·비용·사용한도가 학교별로 분리됨

## 개인정보·보안

- DB에 행 수준 보안(RLS) 적용: 학생은 본인 데이터와 본인이 속한 클래스만 조회 가능
- 세특 초안과 AI 채점 근거는 교사만 열람 (학생에게 비공개)
- 문제의 정답은 학생 브라우저로 아예 내려가지 않음 (채점은 서버에서만 수행)
- 제출 파일은 비공개 저장소에 보관되고 권한 확인 후 한시적 링크로만 열람
- 보안 헤더(CSP 등) 적용, 검색엔진 비노출
- 개인정보처리방침 페이지 제공 (제출물이 AI 분석을 위해 Google Gemini로 전송되는 점 명시)

## 솔직한 한계

- 구버전 `.hwp` 미지원 (`.hwpx`만 — 한글에서 "다른 이름으로 저장" → HWPX)
- 스캔 이미지만 있는 PDF는 텍스트 처리 불가
- 큰 PDF(약 4.5MB 초과)는 업로드가 막힐 수 있음
- 재제출 시 이전 제출물·채점 결과는 덮어써짐(이력 없음)
- AI가 만든 것은 어디까지나 **초안**이며, 교사의 검토가 전제
